## Release 发布说明

### 新特性

- ✨ **丰富 MCP 拓展能力**
  - **解耦的网关设计**：程序仅作为 MCP 网关，负责与云端大模型的 JSON-RPC 消息交互、协议解析及工具生命周期管理。
  - **配置即生效**：支持动态工具配置，修改 `xiaozhi_config.json` 并重启程序即可生效，无需重新编译主程序。
  - **多传输协议支持**：支持 **Subprocess（子进程）**、**HTTP** 和 **TCP Socket** 三种传输方式，可通过配置灵活切换。
  - **双执行模式**：
    - **Sync（对话级同步）**：等待工具执行完成，结果直接返回给大模型（适用于耗时短的工具）。
    - **Background（对话级异步）**：后台执行任务，立刻返回大模型以维持对话流畅性（适用于长耗时工具）。
  - **统一超时与纯异步非阻塞**：所有工具调用均有超时保护（默认 5000ms），底层基于 `tokio` 异步运行时，防止外部脚本假死阻塞系统。
  - 详见 [MCP 功能说明文档](./docs/MCP功能说明.md)
  - 在`examples`目录下增加 MCP 拓展的示例

- **移除 musl 构建支持，新增 uClibc 构建支持**

- **规范 release 和 tag 发布方式**

  当前尚处于架构探索期，会有很多功能更新和迭代，并不稳定，因此 tag 采用v0.x.x 的方式发布，待整体功能趋于稳定后，会采用 v1.x.x 的方式发布。




## 部署说明
Release 中分发的二进制文件基于 GCC8.3 构建，无法在 glibc 版本低于 2.28 的系统上运行，低于 2.28 的系统请参照编译脚本自行编译。其中 uclibc 的二进制文件基于`arm-rockchip830-linux-uclibcgnueabihf`工具链构建，uclibc 比较少见，目前只遇到过 rv1103/rv1106 使用，如后续遇到其他 vendor 的 uclibc 系统，欢迎提交 issue。
